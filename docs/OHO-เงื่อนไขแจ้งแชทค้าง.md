# OHO Chat Watcher — เงื่อนไขการแจ้งเตือน "แชทค้าง"

เอกสารสรุปตรรกะจริงจากโค้ด `scripts/oho-watch.mjs` และ `scripts/oho-shifts.mjs`
(ไว้ให้ AI/คนใหม่เข้าใจว่าระบบตัดสิน "แชทค้าง" และแจ้งเตือนอย่างไร)

## ภาพรวม
มีสคริปต์เฝ้าหน้าเว็บ OHO Chat (`https://app.oho.chat`) ด้วยเบราว์เซอร์ headless (Playwright)
คอยหา "แชทที่ลูกค้าทักแล้วยังไม่มีแอดมินรับ/ตอบ" เกินเวลาที่กำหนด → แคปภาพแถวแชทนั้น →
ส่งเตือนเข้ากลุ่ม Telegram พร้อมแท็กแอดมินเวร + ผู้จัดการ + เจ้าของ

ตรวจทุก **60 วินาที** (poll)

---

## 1) เงื่อนไข "นับว่าเป็นแชทค้าง" (ต้องผ่านทุกข้อ)

แชทหนึ่งจะถูกนับว่า "ค้าง" ก็ต่อเมื่อ:

1. **มีข้อความลูกค้าที่ยังไม่ได้อ่าน** — แถวแชทมี `.message.unread`
   **หรือ** เป็นแชทใหม่ที่รอรับ — มี `.case-status.start`
   - เหตุผล: ถ้าแอดมินรับ/กำลังตอบอยู่ ข้อความล่าสุดจะเป็นของแอดมิน = ไม่ unread → ระบบตัดออกเอง

2. **ไม่ใช่แชทที่ปิด/จบไปแล้ว** — ข้อความล่าสุด (preview) ต้อง **ไม่ขึ้นต้นด้วย**:
   `จบแชท` / `จบเคส` / `ปิดแชท` / `ปิดเคส` / `โอนแชท`
   (พวกนี้คือเคสที่ปิดแล้ว ไม่ใช่ลูกค้าทักค้าง)

3. **อ่านเวลารอได้** — มีตัวจับเวลา (รูปแบบ `HH:MM:SS`) หรือเวลาสัมพัทธ์ไทย
   ("8 นาที", "1 ชั่วโมง", "ไม่กี่วินาที") ถ้าอ่านเวลาไม่ได้ → ข้าม

> พูดง่ายๆ: **ลูกค้าทักค้าง ยังไม่มีคนอ่าน/รับ และแชทยังไม่ปิด**

---

## 2) เกณฑ์เวลา + ระดับความด่วน (escalation)

แจ้งเตือนตาม "เวลาที่รอมาแล้ว":

| ระดับ | เวลารอ | ป้าย |
|---|---|---|
| 🟡 Level 1 | ≥ **3 นาที** (180 วิ) | ค้างยังไม่มีคนรับ |
| 🟠 Level 2 | ≥ **5 นาที** (300 วิ) | ยังไม่รับ เกิน 5 นาที (เตือนซ้ำ) |
| 🔴 Level 3 | ≥ **10 นาที** (600 วิ) | ด่วนมาก! ค้างเกิน 10 นาที |

- แจ้งเตือน **เฉพาะตอนแชท "ข้ามระดับใหม่"** (ระดับสูงกว่าครั้งที่เตือนล่าสุดของแชทนั้น) → ไม่สแปมซ้ำระดับเดิม
- เกณฑ์ 3 นาที ปรับได้ผ่าน env `OHO_THRESHOLD_SECONDS`

---

## 3) กันสแปม / การทยอยแจ้ง

- **รอบแรกที่เริ่มเฝ้า (primed):** บันทึกแชทค้างเดิมทั้งหมดเป็น "รู้แล้ว" ไม่เตือนเดี่ยว
  แล้วสรุปครั้งเดียวว่า "ตอนนี้มีแชทค้างอยู่ N รายการ" → หลังจากนี้เตือนเฉพาะแชทค้าง**ใหม่/ยกระดับ**
- **จำกัดต่อรอบ:** เตือนได้สูงสุด **6 แชท/รอบ** (`OHO_MAX_PER_TICK`) เรียงด่วนสุด (รอนานสุด) ก่อน
  ที่เหลือขึ้นข้อความ "…และมีแชทค้างรออีก N รายการ" แล้วทยอยรอบถัดไป
- แชทที่หายไปจากรายการค้าง (แอดมินรับ/ปิดแล้ว) → ล้างสถานะออก (พร้อมเตือนใหม่ถ้ากลับมาค้างอีก)

---

## 4) แท็กใคร (ตารางเวร — เวลาไทย)

จาก `oho-shifts.mjs` — แท็ก = **เวรตามช่วงเวลา + ผู้จัดการ (พี่หนิง) + เจ้าของ (พี่โด้) เสมอ**

**วันธรรมดา (จันทร์–ศุกร์)**
- เช้า 08:00–17:00 → **พี่น้อง + เนย**
- ดึก 17:00–02:00 → **พี่กร + พี่แพร**

**เสาร์–อาทิตย์**
- เช้า 08:00–16:00 → **พี่น้อง + เนย**
- ดึก 16:00–00:00 → **พี่กร + พี่แพร**

**นอกเวลาข้างบน (ไม่มีเวร)** → แนบหมายเหตุ "นอกเวลาทำการ/ไม่มีเวร — แจ้งผู้จัดการกับพี่โด้ตรง"

หมายเหตุพิเศษที่แนบอัตโนมัติ:
- **ช่วงพัก** (เช้า 11:30–13:30, ดึก 20:30–22:30) → "ช่วงนี้อาจมีคนพัก"
- **ค้างข้ามกะ** (เวรตอนเริ่ม ≠ เวรตอนนี้) → "แชทค้างข้ามกะ — แท็กกะที่รับช่วงต่อ"

> แท็กด้วย Telegram user ID (`tg://user?id=...`) เพราะทีมไม่มี @username — คลิก/เด้งเตือนได้จริง

## 5) หน้าตาข้อความแจ้งเตือน

แคปภาพแถวแชท + แคปชัน:
```
🟡 ค้างเกิน 3 นาที ยังไม่มีคนรับ
📥 ช่องทาง: <ช่องทาง> · ทีม <ทีม>
👤 ลูกค้า: <ชื่อลูกค้า>
⚠️ รอมาแล้ว: <เวลา>
🔗 เปิดแชท: คลิกเปิดแชทนี้
@เวร @พี่หนิง @พี่โด้ รบกวนกดรับแชทด้วยนะคะ
(หมายเหตุ ถ้ามี)
```

## 6) เซสชัน / การตั้งค่า

- ใช้เซสชันเบราว์เซอร์ที่เซฟไว้ (`.oho-session.json`) — สร้างด้วย `npm run oho:auth`
- ถ้าเซสชันหลุด/หมดอายุ → DM เจ้าของ "เซสชัน OHO หมดอายุ รบกวนรัน `npm run oho:auth` ใหม่" แล้วหยุดเตือนชั่วคราว

**ตัวแปร env**
| env | ค่าเริ่มต้น | ความหมาย |
|---|---|---|
| `OHO_URL` | https://app.oho.chat | หน้าเว็บ OHO |
| `OHO_SESSION_PATH` | .oho-session.json | ไฟล์เซสชัน |
| `OHO_ALERT_CHAT_ID` | — | กลุ่ม Telegram ที่ส่งเตือน |
| `OHO_POLL_SECONDS` | 60 | รอบตรวจ (วินาที) |
| `OHO_THRESHOLD_SECONDS` | 180 | เกณฑ์ค้าง (3 นาที) |
| `OHO_MAX_PER_TICK` | 6 | จำนวนเตือนสูงสุดต่อรอบ |

รันด้วย: `npm run oho:watch` (ต้องมีเซสชันก่อน)
