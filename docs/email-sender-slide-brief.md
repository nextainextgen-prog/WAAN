# บรีฟทำสไลด์: ระบบ “ส่ง Email ลูกค้าผ่านแชท” (Email Sender)

> **สำหรับ AI ที่ทำสไลด์:** ข้อมูล + คำสั่งทำสไลด์ **2 หน้า แนวนอน (16:9)** ธีมเดียวกับสไลด์บริษัท (จะแนบภาพตัวอย่างมาด้วย — ยึดสไตล์ตามภาพ: พื้นสว่างฟ้า-ขาว, ตัวอักษรหนา navy, การ์ดโค้งมน, **ไอคอนเส้น ห้ามอิโมจิ**, เส้น timeline จุดๆ ด้านขวา, เลขหน้าจาง)
>
> - **หน้า 1 = รายละเอียด**: ทำอะไร · ทำยังไง · เอาข้อมูลจากไหน · ความปลอดภัย + โรดแมป
> - **หน้า 2 = Flow การทำงาน**: ไดอะแกรมเส้น (สั่ง → ร่าง → ยืนยัน → ส่ง) + จุดตัดสินใจ
>
> โทน: อธิบายให้ผู้จัดการเข้าใจง่าย เน้น “ส่งเมลถึงลูกค้าได้เร็ว + มีตรวจก่อนส่ง + ปลอดภัยไม่ส่งผิด”
> หมายเหตุ: ฟีเจอร์นี้เป็น **แผน/ดีไซน์** (เฟส 1 กำลังจะทำ) — นำเสนอเป็นภาพระบบที่ออกแบบไว้

---

## ระบบนี้คืออะไร (สรุป 2 บรรทัด)

แอดมิน **พิมพ์สั่งในกลุ่ม Telegram** → **“น้องวาน” ดึงข้อมูลลูกค้าจาก Google Sheet + ร่างอีเมลจากเทมเพลตต้นแบบที่บริษัทเซ็ตไว้** → **แสดงตัวอย่างอีเมลเป็นภาพในกลุ่ม + ปุ่มยืนยัน** → แอดมินกด “ส่งเลย” → ยิงออกจาก **Gmail บริษัท** ถึงลูกค้าจริง

จากที่เคยพิมพ์เมลเองทีละฉบับ copy-paste ข้อมูลลูกค้า → เหลือแค่พิมพ์สั่งประโยคเดียว มีตรวจก่อนส่งทุกครั้ง

---

# หน้า 1 — “ทำอะไร / ทำยังไง / เอาข้อมูลจากไหน / ปลอดภัย”

**หัวสไลด์:** ส่งอีเมลถึงลูกค้าอัตโนมัติ — ผ่านแชท
**หัวรอง:** พิมพ์สั่ง → ดึงข้อมูล → ร่างจากเทมเพลต → ตรวจ → ยืนยัน → ส่ง

### บล็อก A — “ทำอะไร” (3–4 ข้อ ทำเป็นการ์ด/ไอคอน)

1. **ส่งเมลถึงลูกค้าให้อัตโนมัติ** — แอดมินไม่ต้องเปิด Gmail พิมพ์เอง แค่สั่งในแชท
2. **ร่างจากเทมเพลตต้นแบบ** — ใช้แบบฟอร์มที่บริษัทเซ็ตไว้ (ต่ออายุ / ทวงชำระ / ต้อนรับลูกค้าใหม่) เติมข้อมูลลูกค้าให้เอง
3. **ตรวจก่อนส่งทุกครั้ง** — โชว์ตัวอย่างอีเมลเป็นภาพในกลุ่ม + ปุ่มยืนยัน กันส่งผิด
4. **ส่งจาก Gmail บริษัท** — เมลไปถึงลูกค้าจากอีเมลทางการของบริษัท

### บล็อก B — “ทำยังไง” (5 สเต็ป)

| # | ขั้นตอน | รายละเอียด |
|---|---------|-----------|
| 1 | **สั่ง (Command)** | แอดมินพิมพ์ในกลุ่ม เช่น “ส่งเมลต่ออายุ shop_abc” หรือ วางอีเมล + เลือกเทมเพลต |
| 2 | **ดึงข้อมูล (Fetch)** | วานค้นลูกค้าใน Google Sheet ด้วย username → ได้ อีเมล · ชื่อร้าน · วันหมดอายุ |
| 3 | **ร่าง (Draft)** | เติมข้อมูลลงเทมเพลตต้นแบบ ({{ชื่อร้าน}} {{วันหมดอายุ}} ...) + AI เกลาให้เนียน |
| 4 | **พรีวิว + ยืนยัน (Confirm)** | เรนเดอร์ภาพอีเมลจริงส่งเข้ากลุ่ม + ปุ่ม [ส่งเลย] [แก้ไข] [ส่งเทสให้ฉัน] [ยกเลิก] |
| 5 | **ส่ง (Send)** | แอดมินกด “ส่งเลย” → ยิงผ่าน Gmail บริษัท → บันทึกทะเบียน → แจ้ง “ส่งแล้ว + เวลา” |

### บล็อก C — ⭐ “เอาข้อมูล/เนื้อหาจากไหน” (จุดที่ผู้จัดการอยากรู้)

- **ข้อมูลลูกค้า = Google Sheet** (ชีตเดียวกับที่ทีมใช้อยู่) — คอลัมน์: อีเมล · username · ชื่อร้าน · วันหมดอายุ ฯลฯ · วานค้นแถวที่ตรงกับ username ที่แอดมินสั่ง
- **เนื้อหาอีเมล = เทมเพลตต้นแบบของบริษัท** — แบบฟอร์มที่เซ็ตไว้ล่วงหน้า มีช่องแทนค่า ({{...}}) วานเติมข้อมูลจริงลงไป (ไม่แต่งเนื้อหาเอง)
- **AI = ช่วยเกลาถ้อยคำให้เนียน** ตามข้อมูลลูกค้าแต่ละราย (ไม่เปลี่ยนสาระ)
- **2 ทางเข้า**: (A) พิมพ์ username → ดึงจาก Sheet · (B) วางอีเมลตรงๆ → ใช้เลย

### บล็อก D — ความปลอดภัย (จุดที่ผู้จัดการต้องมั่นใจ)

- **ต้องยืนยันก่อนส่งเสมอ** — ไม่มีการยิงเมลอัตโนมัติโดยไม่ผ่านสายตาคน
- **ส่งได้เฉพาะคนที่ได้รับอนุญาต** — เฉพาะเจ้าของ/แอดมินที่กำหนดกดปุ่ม “ส่งเลย” ได้
- **ยกเลิกได้ทัน (Undo 10 วิ)** — หลังกดส่ง มีเวลา 10 วินาทีกดยกเลิก กันส่งพลาด
- **กันส่งซ้ำ** — ทะเบียนจำว่าใครได้เมลอะไรเมื่อไหร่ ถ้าจะส่งซ้ำใน 3 วันวานเตือนก่อน
- **ส่งเทสให้ตัวเองก่อนได้** — ปุ่ม “ส่งเทสให้ฉัน” ดูเมลจริงในกล่องตัวเองก่อนยิงลูกค้า
- **ตรวจอีเมลก่อนส่ง** — เช็ก format + reply-to + ลายเซ็นบริษัทให้อัตโนมัติ

### บล็อก E — โรดแมป (ทำเป็นแถบ/ชิป “เฟสถัดไป”)

- **เฟส 1 (กำลังทำ)**: ส่งเดี่ยว · ยืนยันก่อนส่ง · ดึงจาก Sheet · เทมเพลตต้นแบบ
- **เฟส 2**: **Mass campaign** — “ส่งเตือนต่ออายุลูกค้าที่หมดใน 7 วัน” → สแกน Sheet เจอ 23 ราย → ส่งทีละคน personalized → รายงานสำเร็จ/ตกหล่น
- **เฟส 3**: **Reply 2 ทาง** — ลูกค้าตอบเมลกลับ → วานสรุปเข้ากลุ่ม + ร่างคำตอบให้
- **เฟส 4**: ตั้งเวลาส่ง (ไม่ส่งกลางดึก) · อนุมัติ 2 ชั้นสำหรับเมลสำคัญ · แนบเอกสาร PDF อัตโนมัติ

### แถบผลลัพธ์ (ปิดท้ายหน้า 1 — 4 การ์ดสั้น)

- **เร็ว** — พิมพ์สั่งประโยคเดียว ได้เมลถึงลูกค้า
- **ถูกต้อง** — ดึงข้อมูลจริงจาก Sheet ไม่ต้องพิมพ์เอง
- **ปลอดภัย** — ตรวจก่อนส่ง + ยกเลิกได้ + กันส่งซ้ำ
- **ต่อยอดได้** — ขยายเป็นแคมเปญ + ตอบกลับอัตโนมัติ

# หน้า 2 — “Flow การทำงาน” (ไดอะแกรมเส้น)

> ทำเป็น flow ซ้าย→ขวา มีจุดตัดสินใจ (ยืนยัน) กลางเส้น + จุดแตกวน (loop) ที่ “แก้ไข”

```
[แอดมินสั่งในกลุ่ม Telegram]
   ├─ A) พิมพ์ username: “ส่งเมลต่ออายุ shop_abc”
   └─ B) วางอีเมล + เลือกเทมเพลต: “ส่งเมล x@mail.com เทมเพลตต่ออายุ”
        ▼
[ดึงข้อมูลลูกค้าจาก Google Sheet]  (อีเมล · ชื่อร้าน · วันหมดอายุ ตาม username)
        │  ── (ไม่เจอ/ไม่มีอีเมล) ──►  [แจ้งแอดมิน + ขอข้อมูลเพิ่ม]
        ▼
[เติมตัวแปรลงเทมเพลตต้นแบบ + AI เกลาถ้อยคำ]
        ▼
[เรนเดอร์ภาพอีเมลจริง → ส่งพรีวิวเข้ากลุ่ม]
   📧 ถึง: ... · เรื่อง: ... + รูปตัวอย่างอีเมล
        ▼
◇ ปุ่มในแชท: [ส่งเลย] · [แก้ไข] · [ส่งเทสให้ฉัน] · [ยกเลิก]
        ├─[แก้ไข]────► พิมพ์จุดที่แก้ → ร่างใหม่ ──► วนกลับพรีวิว
        ├─[ส่งเทสให้ฉัน]─► ยิงเข้าเมลแอดมินเองก่อน (ดูจริง) ──► วนกลับปุ่ม
        ├─[ยกเลิก]───► จบ (ไม่ส่ง)
        └─[ส่งเลย] (เฉพาะเจ้าของ/ผู้ได้รับอนุญาต)
                ▼
        ◇ กันส่งซ้ำ: เพิ่งส่งเมลนี้ใน 3 วัน? ── (ใช่) ──► [เตือนยืนยันอีกครั้ง]
                ▼ (ผ่าน)
        [Undo 10 วิ: ปุ่มยกเลิกส่ง] ── (กดทัน) ──► [ยกเลิก ไม่ส่ง]
                ▼ (ครบเวลา)
        [ยิงผ่าน Gmail บริษัท] (reply-to + ลายเซ็นบริษัท)
                ▼
        [บันทึกทะเบียนส่ง (ใคร/เมลอะไร/เมื่อไหร่)]
                ▼
        [แจ้งในกลุ่ม “ส่งแล้ว ✅ + เวลา” + แท็กแอดมิน]
```

### เส้นขยาย (เฟส 2 — Mass campaign) — ทำเป็นการ์ดแยก/เส้นประ
```
“ส่งเตือนต่ออายุลูกค้าที่หมดใน 7 วัน”
   → สแกน Sheet หา match ทั้งหมด → เจอ N ราย
   → โชว์ตารางชื่อ+จำนวน → [ยืนยันส่งทั้งหมด]
   → ส่งทีละคน personalized (ชื่อร้าน/วันหมดจริง)
   → รายงาน: ส่งสำเร็จ X / ตกหล่น Y (ไม่มีอีเมล)
```

### หมายเหตุประกอบไดอะแกรม (การ์ดเล็ก)
- แหล่งข้อมูล: **Google Sheet ลูกค้า** (คอลัมน์ อีเมล/username/ชื่อร้าน/วันหมดอายุ)
- เนื้อหา: **เทมเพลตต้นแบบบริษัท** (มีช่องแทนค่า {{...}}) · AI เกลาถ้อยคำ ไม่เปลี่ยนสาระ
- ส่งจาก **Gmail บริษัท** · ทุกฉบับผ่านการยืนยันจากคนก่อนส่ง
