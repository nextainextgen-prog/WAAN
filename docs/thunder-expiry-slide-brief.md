# บรีฟทำสไลด์: ระบบ “ขยายวันหมดอายุ Thunder”

> **สำหรับ AI ที่ทำสไลด์:** ข้อมูล + คำสั่งทำสไลด์ **2 หน้า แนวนอน (16:9)** ธีมเดียวกับสไลด์บริษัท (จะแนบภาพตัวอย่างมาด้วย — ยึดสไตล์ตามภาพ: พื้นสว่างฟ้า-ขาว, ตัวอักษรหนา navy, การ์ดโค้งมน, **ไอคอนเส้น ห้ามอิโมจิ**, เส้น timeline จุดๆ ด้านขวา, เลขหน้าจาง)
>
> - **หน้า 1 = รายละเอียด**: ทำอะไร · ทำยังไง · เอาข้อมูลจากไหน · ความปลอดภัย
> - **หน้า 2 = Flow การทำงาน**: ไดอะแกรมเส้น 2 เฟส (พรีวิว → ยืนยัน → ปรับจริง)
>
> โทน: อธิบายให้ผู้จัดการ/คนไม่เทคนิคเข้าใจง่าย เน้น “ปลอดภัย ไม่แก้ผิด” เพราะเป็นการแก้ข้อมูลลูกค้าจริง

---

## ระบบนี้คืออะไร (สรุป 2 บรรทัด)

แอดมินแค่ **พิมพ์ username ของลูกค้า** ในกลุ่ม Telegram → **“น้องวาน” เข้าไปปรับ “วันที่บอทหมดอายุ” ในระบบหลังบ้าน Thunder ให้เอง** (ตั้งเป็นวัน/เวลาปัจจุบัน = ต่ออายุให้บอทลูกค้ากลับมาใช้งานได้) โดยทำงานแบบ **“แสดงตัวอย่างให้ตรวจก่อน → กดยืนยัน → ค่อยแก้จริง”** กันแก้ผิดคน/ผิดวัน

---

# หน้า 1 — “ทำอะไร / ทำยังไง / เอาข้อมูลจากไหน / ปลอดภัยแค่ไหน”

**หัวสไลด์:** ต่ออายุบอทลูกค้าให้เอง — ผ่านแชท
**หัวรอง:** พิมพ์ username → ตรวจ → ยืนยัน → ระบบปรับหลังบ้านให้อัตโนมัติ

### บล็อก A — “ทำอะไร” (2–3 ข้อ)

1. **ต่ออายุให้ลูกค้าผ่านแชท** — แอดมินไม่ต้องเข้าเว็บหลังบ้านเอง แค่พิมพ์ username
2. **ปรับ “วันที่บอทหมดอายุ” ให้เป็นวันนี้** — ในระบบ Thunder Admin (old.thunder.in.th/admin/service) บอทลูกค้ากลับมาใช้ได้ทันที
3. **แก้เฉพาะร้านที่ถูกต้อง** — เลือกเฉพาะแถว “สาขาหลัก” ที่ username ตรงเป๊ะ ไม่แตะร้านอื่น

### บล็อก B — “ทำยังไง” (2 เฟส — จุดขายคือความปลอดภัย)

**เฟส 1 · ตรวจก่อน (Preview)**
1. น้องวานล็อกอินเข้าหลังบ้าน Thunder (ใช้เซสชันที่เก็บไว้) เปิดหน้า `admin/service`
2. **ค้นหา username** ที่แอดมินพิมพ์มา
3. อ่านตาราง เลือกเฉพาะ **แถว “สาขาหลัก” ที่ username ตรงเป๊ะ**
4. **แคปหน้าจอ 2 รูป ตีกรอบแดง** ที่แถวเป้าหมาย แล้วส่งให้แอดมินตรวจ พร้อม **ปุ่มยืนยัน**
   *(ยังไม่แตะข้อมูลใดๆ)*

**เฟส 2 · ปรับจริง (Execute — หลังกดยืนยันเท่านั้น)**
5. คลิกปุ่มแก้ไข (ดินสอ) ในช่องวันหมดอายุ → เปิดปฏิทิน
6. **ตั้งเป็นวัน/เวลาปัจจุบัน** (ระบบเช็คเดือน/ปีให้ตรงก่อน ถ้าตั้งพลาดจะยกเลิก ไม่บันทึก)
7. กด **“บันทึกข้อมูล”**
8. แคปรูปแถวที่ปรับแล้ว (ตีกรอบแดง) ส่งกลับ + ข้อความ “✅ เรียบร้อย (ปรับ N สาขาหลัก)”

### บล็อก C — ⭐ “เอาข้อมูลจากไหน / แก้อะไร”

อ่านจาก **ตารางในระบบหลังบ้าน Thunder Admin (`/admin/service`)** โดยตรง — จับคอลัมน์เหล่านี้:

| คอลัมน์ที่อ่าน | ใช้ทำอะไร |
|---|---|
| **Username / ชื่อผู้ใช้** | จับคู่ให้ตรงเป๊ะกับที่แอดมินพิมพ์ |
| **ประเภทสาขา** | เลือกเฉพาะ “สาขาหลัก” (ข้ามสาขาอื่น) |
| **วันที่บอทหมดอายุ** | ช่องที่จะแก้ → ตั้งเป็นวัน/เวลาปัจจุบัน |
| **ชื่อร้านค้า / ไอดีร้านค้า** | แสดงในพรีวิวให้แอดมินตรวจ |
| **สถานะ** | ดูว่าร้านนี้ “หมดอายุ” หรือยัง |

- **ตั้งค่าเป็น**: วัน/เวลา**ปัจจุบัน** (ไม่ใช่บวกจำนวนวัน)
- **กรณีลูกค้ามีหลายสาขาหลัก** เลือกได้ 2 แบบ:
  - **เฉพาะที่หมดอายุ** (ค่าเริ่มต้น) — ปรับเฉพาะสาขาที่หมดอายุแล้ว
  - **ทุกสาขา** — ปรับสาขาหลักทั้งหมดของ username นั้น
  - ถ้าไม่มีสาขาไหนหมดอายุเลย → ระบบเตือนก่อน ต้องกดยืนยัน “ปรับทั้งที่ยังไม่หมด” เองถึงจะทำ

### บล็อก D — ความปลอดภัย (จุดที่ผู้จัดการต้องมั่นใจ)

- **ต้องกดยืนยันก่อนเสมอ** — เฟสตรวจไม่แตะข้อมูล จะแก้จริงต่อเมื่อแอดมินกดปุ่มยืนยัน
- **แคป 2 รูปตีกรอบแดง** ให้เห็นชัดว่ากำลังจะแก้ร้านไหน ช่องไหน
- **จับ username ตรงเป๊ะ + สาขาหลักเท่านั้น** — ไม่มีทางแก้ผิดร้าน
- **กันวันผิด** — ก่อนบันทึก ระบบตรวจว่าปฏิทินอยู่เดือน/ปีที่ถูกต้อง ถ้าเพี้ยนจะกดยกเลิก ไม่บันทึก
- **เฉพาะกลุ่มงานนี้เท่านั้น** — ทำได้แค่ในกลุ่มที่ตั้งหน้าที่ “ขยายวันหมดอายุ” และเฉพาะเจ้าของ/คนที่ได้รับอนุญาต · ห้ามเปิด/ค้นเว็บอื่นเด็ดขาด
- **มีระบบดักข้อผิดพลาด**: ไม่พบ username, ไม่มีสาขาหลัก, เซสชันหมดอายุ (แจ้งให้ล็อกอินใหม่) ฯลฯ

### แถบผลลัพธ์ (ปิดท้ายหน้า 1 — 3–4 การ์ดสั้น)

- **เร็ว** — แอดมินไม่ต้องเข้าเว็บหลังบ้านเอง พิมพ์ชื่อเดียวจบ
- **แม่น** — แก้เฉพาะร้านที่ถูกต้อง มีรูปยืนยันก่อน–หลัง
- **ปลอดภัย** — ต้องยืนยันก่อน + กันวันผิด/คนผิด
- **ตรวจสอบย้อนหลังได้** — บันทึกทุกครั้งที่ปรับ (ใคร/ร้านไหน/เดิมเป็นวันอะไร)

# หน้า 2 — “Flow การทำงาน” (ไดอะแกรมเส้น 2 เฟส)

> ทำเป็น flow ซ้าย→ขวา แบ่งชัดเป็น **2 โซน**: โซนบน = “ตรวจก่อน (Preview)” · โซนล่าง = “ปรับจริง (Execute)” มีจุดตัดสินใจ (แอดมินกดยืนยัน) คั่นกลาง

### เฟส 1 — ตรวจก่อน (สีฟ้าอ่อน)

```
[แอดมินพิมพ์ username ในกลุ่ม Telegram “ขยายวันหมดอายุ”]
        ▼
[น้องวานล็อกอินหลังบ้าน Thunder → เปิด admin/service]
        ▼
[ค้นหา username → อ่านตาราง]
        ▼
[เลือกเฉพาะแถว “สาขาหลัก” ที่ username ตรงเป๊ะ]
        ▼
[แคป 2 รูป ตีกรอบแดง: (ซ้าย) ยูสเซอร์/สาขา · (ขว) วันหมดอายุ+สถานะ]
        ▼
[ส่งพรีวิว + ปุ่มยืนยันเข้ากลุ่ม]
```

### จุดตัดสินใจ (คั่นกลาง — ไอคอนคน/มือกดปุ่ม)

```
◇ แอดมินตรวจรูปแล้ว “กดยืนยัน” ?
   ├─ ยกเลิก → จบ (ไม่แตะข้อมูล)
   └─ ยืนยัน (เลือก scope: เฉพาะที่หมดอายุ / ทุกสาขา) ▼
```

### เฟส 2 — ปรับจริง (สีเข้ม/เน้น)

```
[คลิกปุ่มแก้ไข (ดินสอ) ในช่องวันหมดอายุ → เปิดปฏิทิน]
        ▼
[ตั้งวัน/เวลาปัจจุบัน]  ──(ถ้าตั้งเดือน/ปีเพี้ยน)──►  [ยกเลิก ไม่บันทึก] (กันวันผิด)
        ▼ (ถูกต้อง)
[กด “บันทึกข้อมูล”]
        ▼
[แคปรูปแถวที่ปรับ (ตีกรอบแดง)]
        ▼
[บันทึกกิจกรรม + ส่งผล “✅ เรียบร้อย ปรับ N สาขาหลัก” + 2 รูป]
```

### หมายเหตุประกอบไดอะแกรม (ทำเป็นการ์ดเล็ก/เชิงอรรถ)
- แหล่งข้อมูล: ตาราง **Thunder Admin → /admin/service** (คอลัมน์ username · ประเภทสาขา · วันที่บอทหมดอายุ · ชื่อร้าน · ไอดีร้าน · สถานะ)
- ใช้บอทเปิดเบราว์เซอร์ทำแทนคน (ล็อกอินด้วยเซสชันที่เก็บไว้)
- เฟสตรวจ **ไม่แตะข้อมูล** · แก้จริงหลังกดยืนยันเท่านั้น
