# บรีฟทำสไลด์: ระบบ “ทำเอกสารคืนเงิน (บันทึกข้อความคืนเงินหัก ณ ที่จ่าย)”

> **สำหรับ AI ที่ทำสไลด์:** ข้อมูล + คำสั่งทำสไลด์ **2 หน้า แนวนอน (16:9)** ธีมเดียวกับสไลด์บริษัท (จะแนบภาพตัวอย่างมาด้วย — ยึดสไตล์ตามภาพ: พื้นสว่างฟ้า-ขาว, ตัวอักษรหนา navy, การ์ดโค้งมน, **ไอคอนเส้น ห้ามอิโมจิ**, เส้น timeline จุดๆ ด้านขวา, เลขหน้าจาง)
>
> - **หน้า 1 = รายละเอียด**: ทำอะไร · ทำยังไง · เอาข้อมูลจากไหน · ความปลอดภัย + การเซ็นอนุมัติ
> - **หน้า 2 = Flow การทำงาน**: ไดอะแกรมเส้น (สั่ง → ร่าง → เซ็น 2 ชั้น)
>
> โทน: อธิบายให้ผู้จัดการเข้าใจง่าย เน้น “ออกเอกสารทางการได้เร็ว + ปลอดภัย + มีขั้นเซ็นอนุมัติ”

---

## ระบบนี้คืออะไร (สรุป 2 บรรทัด)

แอดมิน **พิมพ์สั่ง + แนบหลักฐาน** ในกลุ่ม Telegram → **“น้องวาน” ออก “บันทึกข้อความขออนุมัติคืนเงินส่วนต่างหัก ณ ที่จ่าย”** เป็นไฟล์ PDF ทางการให้ทันที (ล็อกรหัสผ่าน) แล้วส่งเข้าขั้น **ตรวจ + เซ็นอนุมัติ 2 ชั้น** — จากที่เคยพิมพ์เอกสารเองทีละใบ เหลือแค่พิมพ์สั่งประโยคเดียว

**คืนเงิน 3 ส่วนรวมกัน:** หัก ณ ที่จ่าย + ส่วนเกินที่ชำระ + ส่วนลด

---

# หน้า 1 — “ทำอะไร / ทำยังไง / เอาข้อมูลจากไหน / ปลอดภัย + เซ็นอนุมัติ”

**หัวสไลด์:** ออกเอกสารคืนเงินอัตโนมัติ — ผ่านแชท
**หัวรอง:** พิมพ์สั่ง → ได้ PDF ทางการ → ตรวจ → เซ็นอนุมัติ 2 ชั้น

### บล็อก A — “ทำอะไร” (2–3 ข้อ)

1. **ออกบันทึกข้อความคืนเงินให้อัตโนมัติ** — เอกสารภายในทางการของบริษัท (มีเลขที่เอกสาร หัวบริษัท ช่องเซ็น)
2. **รวมยอดคืนให้ครบ** — หัก ณ ที่จ่าย + ส่วนเกิน + ส่วนลด คำนวณให้เสร็จ
3. **แนบหลักฐานท้ายเอกสาร** — สลิป / 50 ทวิ / bookbank ที่แอดมินแนบมา ต่อเป็นหน้าถัดไปอัตโนมัติ

### บล็อก B — “ทำยังไง” (5 สเต็ป)

1. **แอดมินพิมพ์สั่ง + แนบไฟล์หลักฐาน** ในกลุ่ม → ระบบจับคำว่า “คืนเงิน / หัก ณ ที่จ่าย / ส่วนต่าง” อัตโนมัติ (ยังไม่มีไฟล์ก็รอไฟล์ให้)
2. **AI อ่านข้อความสั่งงาน** แล้วสกัดเป็นข้อมูลมีโครงสร้าง (ชื่อลูกค้า/ยอด/บัญชี/วันที่ ฯลฯ) + คำนวณยอดคืน
3. **สร้างเอกสาร** — เติมข้อมูลลงเทมเพลตทางการ แล้วเรนเดอร์เป็น **PDF A4** พร้อมแนบหลักฐานเป็นหน้าถัดไป
4. **ล็อกไฟล์ด้วยรหัสผ่าน** แล้วส่งร่างเข้ากลุ่ม (ปิดข้อมูลอ่อนไหวไว้) พร้อมปุ่ม **[เซ็นเลย] [แก้ไข]**
5. **ตรวจ → เซ็นอนุมัติ** — เจ้าของตรวจแล้วกดเซ็น → ระบบใส่ลายเซ็น + แท็กผู้จัดการให้เซ็นต่อ

### บล็อก C — ⭐ “เอาข้อมูลจากไหน” (จุดที่ผู้จัดการอยากรู้)

- **สกัดจาก “ข้อความที่แอดมินพิมพ์สั่ง” เท่านั้น** — AI ถูกสั่ง **ห้ามอ่านข้อมูลจากรูป/ไฟล์แนบเด็ดขาด** (สลิป/50 ทวิ เป็นแค่ “เอกสารประกอบ” ต่อท้าย ไม่เอามาเป็นตัวเลขในเอกสาร) → กันอ่านเลขผิดจากภาพ
- ข้อมูลที่สกัด: ชื่อลูกค้า/บริษัท · ประเภทบริการ · แพ็กเกจ/จำนวนเดือน · ราคาที่ต้องชำระ · ยอดที่ชำระเข้ามา · อัตรา/ยอดหัก ณ ที่จ่าย · ส่วนลด · ส่วนเกิน · ธนาคาร/เลขบัญชี/ชื่อบัญชี · วันที่เติมเงิน
- **เทมเพลตเอกสาร** = แบบฟอร์มบันทึกข้อความทางการของบริษัท (หัวบริษัท เลขผู้เสียภาษี เนื้อความล็อกตามต้นฉบับ เติมเฉพาะช่องว่าง) + ฟอนต์ราชการ (Sarabun) + โลโก้บริษัท
- **ลายเซ็น** = ไฟล์ภาพลายเซ็นที่เตรียมไว้ จะแสดงเมื่อกด “เซ็น” เท่านั้น (ร่างยังไม่มีลายเซ็น)

### บล็อก D — ความปลอดภัย

- **ล็อก PDF ด้วยรหัสผ่าน (เข้ารหัส AES-256)** — เปิดไฟล์ต้องใส่รหัส (รหัสกำหนดไว้ในระบบ ไม่โชว์ในแชท)
- **ปิดข้อมูลอ่อนไหวในแชท** — ชื่อลูกค้า/ยอดคืน/ยอดหัก เบลอไว้ (แตะเพื่อดู) · รหัสเปิดไฟล์แสดงแบบซ่อน ไม่พิมพ์เลขจริงในแชท
- **แจ้งให้ทำภายใน 24 ชม.** ก่อนประวัติแชทถูกลบอัตโนมัติ
- **จำกัดสิทธิ์** — สั่งออก/แก้ได้เฉพาะคนที่ได้รับอนุญาตในกลุ่มที่ผูกไว้ · **กดเซ็นได้เฉพาะเจ้าของ**
- ชื่อไฟล์สั้น อ่านง่าย + มีเลขที่เอกสารกำกับทุกใบ

### บล็อก E — เซ็นอนุมัติ 2 ชั้น (จุดที่ผู้จัดการต้องรู้)

1. ระบบออกร่าง → **แท็กเจ้าของ** ให้ตรวจก่อน
2. เจ้าของกด **“เซ็นเลย”** → ระบบเรนเดอร์ใหม่ใส่ลายเซ็น → ส่งไฟล์ (ลงนามแล้ว)
3. ระบบ **แท็กผู้จัดการฝ่ายบริการลูกค้า** ให้เซ็นชั้นที่ 2 → ส่งต่อฝ่ายบัญชี
   *(เอกสารมี 3 ช่องเซ็น: หัวหน้าฝ่าย CS → ผู้จัดการฝ่าย CS → ผู้อนุมัติ/ผู้บริหาร)*
- ปุ่ม **“แก้ไข”**: พิมพ์บอกจุดที่อยากแก้ (วันที่/ยอด/ธนาคาร/ที่อยู่) → ระบบออกร่างใหม่ **เลขที่เอกสารเดิม ไฟล์แนบเดิม**

### แถบผลลัพธ์ (ปิดท้ายหน้า 1 — 4 การ์ดสั้น)

- **เร็ว** — พิมพ์ประโยคเดียวได้เอกสารทางการทันที
- **ถูกต้อง** — คำนวณยอดคืนให้ + ตรวจข้อมูลก่อนออก
- **ปลอดภัย** — ไฟล์ล็อกรหัส + ปิดข้อมูลในแชท
- **ครบขั้นตอน** — มีเซ็นอนุมัติ 2 ชั้นในตัว

# หน้า 2 — “Flow การทำงาน” (เส้นเดียว ตั้งแต่สั่งจนเซ็นเสร็จ)

> ทำเป็น flow ซ้าย→ขวา มีจุดแตกวน (loop) ที่ “แก้ไข” และจุดเซ็น 2 ชั้นท้ายเส้น

```
[แอดมินพิมพ์สั่ง + แนบไฟล์หลักฐาน (กลุ่ม Telegram)]
        │  (ระบบจับคำว่า “คืนเงิน/หัก ณ ที่จ่าย/ส่วนต่าง”)
        ▼
[ยังไม่มีไฟล์?] ──► รอไฟล์ (เปิดหูรับ) ──┐
        │ มีไฟล์แล้ว                      │
        ▼ ◄───────────────────────────────┘
[AI สกัดข้อมูลจาก “ข้อความแอดมิน” เท่านั้น + คำนวณยอดคืน]
        ▼
[เติมลงเทมเพลตทางการ → เรนเดอร์ PDF A4 + แนบหลักฐานเป็นหน้าถัดไป]
        ▼
[ล็อกไฟล์ด้วยรหัสผ่าน (AES-256)]
        ▼
[ส่งร่างเข้ากลุ่ม (ปิดข้อมูล spoiler) + ปุ่ม [เซ็นเลย] [แก้ไข] — แท็กเจ้าของตรวจ]
        │
        ├──[แก้ไข]──► พิมพ์จุดที่แก้ → ออกร่างใหม่ (เลขเอกสารเดิม) ──► วนกลับส่งร่าง
        │
        └──[เซ็นเลย] (เฉพาะเจ้าของ)
                ▼
        [ใส่ลายเซ็น → เรนเดอร์ใหม่ (ลงนามแล้ว) → ล็อกไฟล์]
                ▼
        [ส่งไฟล์ลงนามแล้ว + แท็กผู้จัดการฝ่าย CS ให้เซ็นชั้น 2]
                ▼
        [ผู้จัดการเซ็นต่อ → ส่งต่อฝ่ายบัญชี]
```

### หมายเหตุประกอบไดอะแกรม (การ์ดเล็ก)
- **แหล่งข้อมูล** = ข้อความสั่งของแอดมิน (ไม่ใช่ตัวเลขจากภาพ) · หลักฐานที่แนบ = เอกสารประกอบต่อท้ายเท่านั้น
- **คืนเงิน** = หัก ณ ที่จ่าย + ส่วนเกิน + ส่วนลด
- **เซ็น 2 ชั้น** = เจ้าของ → ผู้จัดการฝ่าย CS (เอกสารมี 3 ช่องเซ็น)
- ไฟล์ล็อกรหัส + เอกสารมีเลขที่กำกับ
