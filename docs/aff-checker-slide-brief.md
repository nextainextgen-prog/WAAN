# บรีฟทำสไลด์: ระบบ “ตรวจเอกสาร AFF (ใบสำคัญรับเงินค่าคอมฯ Affiliate)”

> **สำหรับ AI ที่ทำสไลด์:** ข้อมูล + คำสั่งทำสไลด์ **2 หน้า แนวนอน (16:9)** ธีมเดียวกับสไลด์บริษัท (จะแนบภาพตัวอย่างมาด้วย — ยึดสไตล์ตามภาพ: พื้นสว่างฟ้า-ขาว, ตัวอักษรหนา navy, การ์ดโค้งมน, **ไอคอนเส้น ห้ามอิโมจิ**, เส้น timeline จุดๆ ด้านขวา, เลขหน้าจาง)
>
> - **หน้า 1 = รายละเอียด**: ทำอะไร · ตรวจอะไร · ทำยังไง · เอาข้อมูลจากไหนมาเทียบ
> - **หน้า 2 = Flow การทำงาน**: ไดอะแกรม 2 เส้นทาง (โหมดตรวจ / โหมดวานสร้างเอง)
>
> โทน: อธิบายให้ผู้จัดการเข้าใจง่าย เน้น “ตรวจแม่นจาก 3 แหล่ง + ปิดข้อมูลอ่อนไหว”

---

## ระบบนี้คืออะไร (สรุป 2 บรรทัด)

ระบบตรวจและออก **“ใบสำคัญรับเงิน” ค่าคอมมิชชั่น Affiliate** อัตโนมัติ — แอดมินแนบเอกสารในกลุ่ม Telegram แล้ว **“น้องวาน” สกัดข้อมูล เทียบกับ 3 แหล่ง (เอกสาร / ชีตลูกค้า / ระบบหลังบ้าน) + ตรวจการคำนวณภาษี** แล้วสรุปว่า **“พร้อมอนุมัติ” หรือ “ต้องตรวจทาน”** พร้อมปิดข้อมูลอ่อนไหวไว้ในแชท

**ทำได้ 2 โหมด:** (1) ตรวจเอกสารที่แอดมินส่ง · (2) วานสร้างใบสำคัญรับเงินขึ้นเอง แล้วตรวจเอง รอเจ้าของกดอนุมัติ

---

# หน้า 1 — “ทำอะไร / ตรวจอะไร / ทำยังไง / เทียบจากไหน”

**หัวสไลด์:** ตรวจใบสำคัญรับเงิน Affiliate อัตโนมัติ
**หัวรอง:** สกัด → เทียบ 3 แหล่ง → ตรวจภาษี → สรุปผ่าน/ไม่ผ่าน

### บล็อก A — “ตรวจอะไรบ้าง” (5 กลุ่ม ทำเป็นการ์ด/ไอคอน)

1. **ตัวตน** — ชื่อ-สกุล · เลขผู้เสียภาษี · ที่อยู่
2. **บัญชีรับเงิน** — ธนาคาร · เลขบัญชี
3. **ยอดเงิน** — ยอดจ่ายจริง/สุทธิตรงกับที่ลูกค้าถอน
4. **การคำนวณในเอกสาร** — ยอดเต็ม − หักภาษี = ยอดสุทธิ (ต้องบวกลบถูก)
5. **อัตราหักภาษี** — หัก ณ ที่จ่าย = **3%** ของยอดเต็ม

### บล็อก B — “ทำยังไง” (โหมดตรวจ — 4 สเต็ป)

1. **แอดมินแนบ PDF + ข้อความสรุป** ในกลุ่ม → ระบบจับว่าเป็นเอกสาร AFF อัตโนมัติ
2. **สกัดข้อมูล** — อ่านข้อความในไฟล์ + ให้ AI ช่วยจัดเป็นข้อมูลมีโครงสร้าง (ชื่อ/เลขภาษี/ยอด/บัญชี ฯลฯ)
3. **เทียบ 3 แหล่ง + ตรวจการคำนวณ** — จับคู่ข้อมูลจากเอกสาร ชีต และระบบหลังบ้าน แล้วเช็กเลขให้ตรงกัน
4. **สรุปผล** — reply ในกลุ่ม บอก “พร้อมอนุมัติ ✅” หรือ “ต้องตรวจทาน ❌” + ภาพตารางเทียบ

### บล็อก C — ⭐ “เอาข้อมูลจากไหนมาเทียบ” (3 แหล่ง — จุดที่ผู้จัดการอยากรู้)

ระบบไม่ได้เชื่อเอกสารอย่างเดียว แต่ **เทียบไขว้ 3 แหล่ง**:

| แหล่ง | ได้อะไร | ใช้เป็นตัวตัดสิน |
|---|---|---|
| **1. เอกสาร (PDF)** | ค่าที่พิมพ์อยู่ในใบสำคัญรับเงิน | ตั้งต้นเทียบ |
| **2. ชีตลูกค้า AFF** (Google Sheet บน Drive) | ชื่อจริง · เลขผู้เสียภาษี · ที่อยู่ตามบัตร · ธนาคาร · เลขบัญชี | **ยึดเรื่องตัวตน** (ชื่อ/เลขภาษี/ที่อยู่) |
| **3. ระบบหลังบ้าน Thunder** (`/admin/affiliate`) | ยอดถอนจริง · บัญชีที่กดถอน · สถานะ + แคปหน้าจอ | **ยึดเรื่องบัญชี + ยอดเงินจริง** |

- **เทียบแบบยืดหยุ่น** (ไม่ต้องตรงเป๊ะทุกตัวอักษร): ตัดคำนำหน้าชื่อ · ที่อยู่เทียบเฉพาะเลขบ้าน+อำเภอ+จังหวัด (ยอมพิมพ์ต่างเล็กน้อย) · เลขบัญชี/ภาษีเทียบเฉพาะตัวเลข · ธนาคารตัดคำว่า “ธนาคาร/จำกัด/มหาชน”
- **เกณฑ์ผ่านต่อฟิลด์**: ต้องมี **อย่างน้อย 2 แหล่งตรงกัน**
- **เลือกแถวให้ตรงรายการ**: ใช้วันที่/ยอดจาก noti ที่ระบบแจ้ง เพื่อหยิบแถวถอนที่ถูกต้อง (กันหยิบผิดเมื่อมีหลายรายการ)

> **บทบาทของ AI:** ใช้ช่วย **“อ่าน/สกัดข้อมูลจากไฟล์” เท่านั้น** ส่วนการตัดสินผ่าน/ไม่ผ่านเป็นการคำนวณตรงไปตรงมา (ไม่ให้ AI เดา) — แม่นและตรวจสอบได้

### บล็อก D — ผลลัพธ์ + ความปลอดภัยข้อมูล

- **ปิดข้อมูลอ่อนไหวในแชท** — ชื่อ/เลขภาษี/ที่อยู่/เลขบัญชี/ยอดเงิน เบลอไว้ (แตะเพื่อเปิดดู) ป้ายกำกับกับ ✅/❌ เห็นปกติ
- **มีภาพประกอบ** — ตารางเทียบ 4 คอลัมน์ (ข้อมูล / เอกสาร / ชีต / ระบบ) + ภาพจากระบบหลังบ้าน + สำเนาบัตรตัวตน
- **แท็กอัตโนมัติ** — ผ่าน → แท็กผู้รับผิดชอบว่า “ใช้ข้อมูลได้เลย” · ไม่ผ่าน → แท็กแอดมินที่ส่งให้ “ตรวจทานอีกครั้ง”

### (โหมดที่ 2) วานสร้างใบเอง + ตรวจเอง

เมื่อระบบธันเดอร์แจ้ง noti “กำลังรออนุมัติ” เข้ากลุ่ม → **น้องวานสร้างใบสำคัญรับเงินให้เองอัตโนมัติ** จากโปรไฟล์ลูกค้าที่เก็บไว้ + ยอดถอนจริงจากระบบ (คิดยอดเต็มจากสุทธิ ÷ 0.97, หัก 3%) → ออกไฟล์ PDF จากเทมเพลต + แนบสำเนาบัตร → ตรวจเอง → ส่งร่างพร้อมปุ่ม **[อนุมัติ ✅] [แก้ไข]** ให้เจ้าของกด

### แถบผลลัพธ์ (ปิดท้ายหน้า 1 — 4 การ์ดสั้น)

- **แม่นยำ** — เทียบไขว้ 3 แหล่ง ไม่เชื่อเอกสารอย่างเดียว
- **เร็ว** — ตรวจเสร็จในไม่กี่นาที แทนการไล่เช็กมือ
- **ปลอดภัย** — ปิดข้อมูลอ่อนไหวในแชท
- **ครบวงจร** — ตรวจได้ + ออกใบให้เองได้

# หน้า 2 — “Flow การทำงาน” (2 เส้นทาง)

> ทำเป็นไดอะแกรม **2 แทร็ก**: แทร็กบน = โหมดตรวจ (แอดมินส่งเอกสาร) · แทร็กล่าง = โหมดวานสร้างเอง มีจุดร่วมคือ “เทียบ 3 แหล่ง” และ “รายงานเข้ากลุ่ม”

### แทร็ก A — โหมดตรวจเอกสาร

```
[แอดมินแนบ PDF + ข้อความ “ยูสเซอร์…” ในกลุ่ม Telegram]
        ▼  (ระบบจับว่าเป็นเอกสาร AFF)
[สกัดข้อมูลจากไฟล์]  (AI ช่วยอ่าน + ยึด regex จากข้อความแอดมิน)
        ▼
┌─────────────── เทียบ 3 แหล่ง ───────────────┐
│  เอกสาร PDF                                  │
│  ชีตลูกค้า (Google Sheet)  →  ยึด “ตัวตน”     │
│  ระบบหลังบ้าน Thunder      →  ยึด “บัญชี+ยอด” │
└──────────────────────────────────────────────┘
        ▼
[ตรวจการคำนวณ: ยอดเต็ม−หัก 3% = สุทธิ · ยอดตรงระบบ]
        ▼
[เรนเดอร์ภาพ: ตารางเทียบ 4 คอลัมน์ + ภาพระบบ + หน้าบัตร]
        ▼
[รายงานเข้ากลุ่ม (ปิดข้อมูลด้วย spoiler) + สรุปผ่าน/ไม่ผ่าน]
        ▼
[แท็ก: ผ่าน→ผู้รับผิดชอบ · ไม่ผ่าน→ผู้ส่ง]  +  react ✅/⚠️
```

### แทร็ก B — โหมดวานสร้างเอง

```
[ระบบธันเดอร์แจ้ง noti “กำลังรออนุมัติ” เข้ากลุ่ม AFF]
        ▼
[น้องวานอ่าน noti (username/วันที่/ยอด)]
        ▼
[ดึงโปรไฟล์ลูกค้า (คลังข้อมูลใน Obsidian)]  ──(ไม่เจอ)──► [ขอหน้าข้อมูลลูกค้าใหม่]
        ▼
[ดึงยอด+บัญชีจริงจากระบบหลังบ้าน → คิดยอด (สุทธิ, เต็ม=สุทธิ/0.97, หัก 3%)]
        ▼
[สร้างใบสำคัญรับเงิน PDF (วางทับเทมเพลต + แนบสำเนาบัตร หน้า 2)]
        ▼
[ตรวจเอง (ใช้ flow เทียบ 3 แหล่งเดียวกับแทร็ก A)]
        ▼
[ส่งร่าง + ภาพ + ไฟล์ + สรุป + ปุ่ม [อนุมัติ ✅] [แก้ไข] (แท็กเจ้าของ)]
        ├─[แก้ไข]─► พิมพ์คำสั่งแก้ (วันที่/ยอด/ธนาคาร/ที่อยู่) → สร้างใหม่ (วนกลับ)
        └─[อนุมัติ]─► ตอบกลับ noti ระบบ + แนบไฟล์จริง + แท็ก “อนุมัติแล้ว ✅”
```

### หมายเหตุประกอบไดอะแกรม (การ์ดเล็ก)
- 3 แหล่งเทียบ = เอกสาร · ชีตลูกค้า (Google Drive) · ระบบหลังบ้าน Thunder (`/admin/affiliate`)
- AI = ช่วยอ่าน/สกัดไฟล์เท่านั้น · การตัดสินผ่าน/ไม่ผ่าน = คำนวณตรง ไม่เดา
- โปรไฟล์ลูกค้าเก็บเป็นคลังข้อมูล (Obsidian) · ใบสำคัญรับเงินออกจากเทมเพลตทางการ

## ภาคผนวก — ธีม/สไตล์ (ให้ตรงกับสไลด์บริษัท)

**สี (hex):** Navy `#0A2F5C` · Blue `#1C5FA0` · Brand `#2B7CC9` · Sky `#79B0E5` · Ice `#C8DDF1` · พื้น `#F4F8FD`/ขาว `#FFFFFF` · เส้น `#E3ECF6` · muted `#5E6E84` · จาง `#9AA9BC` · เขียว(ผ่าน) `#22A06B` · แดง(ไม่ผ่าน) `#D9544D` · ทอง(เน้น) `#E3A93C` · กล่องเข้ม `#15437A → #0A2F5C`

**ฟอนต์:** หัวเรื่อง/ตัวเลข = Archivo (800–900) · เนื้อไทย = IBM Plex Sans Thai · โค้ด = JetBrains Mono

**สไตล์:** พื้น gradient ฟ้าอ่อน · การ์ดขาวโค้ง ~16px · ชิปแคปซูล · หัวข้อขีดใต้สั้นสีฟ้า · เลขหน้าจาง · **ไอคอนเส้น ห้ามอิโมจิ** · ใช้ ✅/❌ เป็นไอคอน/จุดสี ไม่ใช่อิโมจิ

**ตัวเลข/ข้อเท็จจริงอ้างอิง:** หัก ณ ที่จ่าย 3% · ผ่านต่อฟิลด์ต้อง ≥2 แหล่งตรงกัน · เทียบ 3 แหล่ง (เอกสาร/ชีต/ระบบ) · ตัวตนยึดชีต · บัญชี+ยอดยึดระบบ · ปิดข้อมูลอ่อนไหวด้วย spoiler · ออกใบสำคัญรับเงินจากเทมเพลตทางการ

**หมายเหตุเทคนิค (ไม่ต้องขึ้นสไลด์):** สกัดไฟล์ด้วย Claude (subscription ไม่ใช้ API key) · ดึงระบบหลังบ้านด้วย Playwright (session จาก `npm run thunder:auth`) · ตรรกะเทียบเป็น deterministic (normalize + Levenshtein)
