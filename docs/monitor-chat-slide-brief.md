# บรีฟทำสไลด์: ระบบ “Monitor Chat — Thunder / Easy”

> **สำหรับ AI ที่ทำสไลด์:** นี่คือข้อมูล + คำสั่งทำสไลด์ **2 หน้า แนวนอน (16:9)** ธีมเดียวกับสไลด์บริษัท (จะแนบภาพตัวอย่างสไลด์มาด้วย — ให้ยึดสไตล์ตามภาพนั้น: พื้นสว่างฟ้า-ขาว, ตัวอักษรหนา navy, การ์ดโค้งมน, ไอคอนเส้น (line icon) **ห้ามใช้อิโมจิ**, มีเส้น timeline จุดๆ ด้านขวา, เลขหน้าจาง).
>
> - **หน้า 1 = รายละเอียด**: ระบบนี้ทำอะไร ทำยังไง และ “เอาข้อมูลจากไหนมาตอบลูกค้า”
> - **หน้า 2 = Flow การทำงาน**: ไดอะแกรมเส้น flow ซ้าย→ขวา + ลูปป้อนกลับ
>
> โทน: อธิบายให้ **ผู้จัดการ / คนไม่เทคนิค** เข้าใจง่ายที่สุด สั้น กระชับ เป็นภาษาคน

---

## ระบบนี้คืออะไร (สรุป 2 บรรทัด)

“Monitor Chat” คือระบบให้บอทผู้ช่วยชื่อ **“น้องวาน”** คอย **เฝ้าดูแชทลูกค้าทุกช่องทางแทนคน ตลอด 24 ชม.** เจอแชทที่ลูกค้ารอนานหรือแอดมินลืมปิด จะ **เตือนเข้ากลุ่ม Telegram กลุ่มเดียว** พร้อมบอกว่าใครต้องรับ แนบรูปหน้าจอ และ **AI ร่างข้อความตอบลูกค้าให้พร้อมก๊อปส่ง**

ครอบคลุม **2 บริษัท (Thunder Solution × EasySlip) · หลายบริการ**: Thunder Bot, Thunder API, EasySlip, EasyCRM, BoostSMS

# หน้า 1 — “ระบบทำอะไร / ทำยังไง / เอาข้อมูลตอบลูกค้าจากไหน”

**หัวสไลด์ (Title):** ระบบเฝ้าแชทลูกค้าอัตโนมัติ — Monitor Chat
**หัวรอง (Subtitle):** เฝ้า 3 แพลตฟอร์ม · เตือนที่เดียว · AI ช่วยร่างคำตอบ

### บล็อก A — “ทำอะไร” (4 ข้อ ทำเป็นการ์ด/ไอคอน)

1. **เฝ้าแชทให้ตลอด 24 ชม.** — คอยดูทุกช่องทางแทนแอดมิน ไม่มีวันลืม ไม่มีวันหยุด
2. **จับแชทที่ต้องรีบ** — ลูกค้าทักแล้วยังไม่มีคนรับ หรือแอดมินรับแล้วลืมปิด
3. **เตือนเข้าที่เดียว** — ส่งการ์ดแจ้งเตือนเข้ากลุ่ม Telegram แยกห้องตามแบรนด์ + แท็กคนที่ต้องรับ
4. **ช่วยตอบลูกค้า** — AI ร่างข้อความตอบให้พร้อมก๊อปส่ง แอดมินแค่ตรวจแล้วส่ง

### บล็อก B — “ทำยังไง” (ขั้นตอนย่อ 5 สเต็ป)

| # | ขั้นตอน | รายละเอียด |
|---|---------|-----------|
| 1 | **เฝ้าดู (Watch)** | บอทเปิดเบราว์เซอร์ล็อกอินเข้า OHO / Facebook / LINE (ใช้เซสชันที่บันทึกไว้) แล้ว **สแกนทุก 60 วินาที** |
| 2 | **ตรวจจับ (Detect)** | เจอ 2 กรณี: (ก) ลูกค้าทักค้าง **รอเกิน 3 นาที** ยังไม่มีคนรับ (ข) แอดมินรับแล้ว **ไม่ปิดเกิน 15 นาที** |
| 3 | **อ่าน + แคป (Read)** | เปิดห้องแชทจริง อ่านข้อความล่าสุด + สถานะ + **แคปหน้าจอ** แล้วจำแนกว่าเป็นแบรนด์ไหน |
| 4 | **AI ร่างคำตอบ (Draft)** | ส่งบทสนทนาให้ AI (Claude) ร่างข้อความตอบลูกค้า → ได้ข้อความพร้อมก๊อป |
| 5 | **แจ้งเตือน (Alert)** | ส่งการ์ดเข้ากลุ่ม Telegram ห้องตามแบรนด์ + แท็กแอดมิน “เวร” + ปุ่มเปิดแชท |

### บล็อก C — ⭐ “เอาข้อมูลจากไหนมาตอบลูกค้า” (จุดที่ผู้จัดการอยากรู้)

AI ไม่ได้มั่วตอบเอง — มัน **ดึงความรู้จากคลังความรู้ของบริษัท (Knowledge Base)** ที่เก็บไว้ใน Obsidian แล้วเอามาประกอบกับบทสนทนาจริงของลูกค้า ก่อนร่างคำตอบ แหล่งข้อมูล 3 ชั้น:

1. **เอกสารสินค้าแต่ละบริการ** — คู่มือ/รายละเอียด/แพ็กเกจ/วิธีใช้ ของ Thunder Bot·API, EasySlip, EasyCRM, BoostSMS (แยกโฟลเดอร์ต่อบริการ)
2. **ข้อมูลบริษัท (company-info)** — ข้อมูลกลางที่ใช้ร่วมกันทุกบริการ
3. **ตัวอย่างคำตอบที่เคยใช้จริง (chat-replies)** — สคริปต์/โทนการตอบที่แอดมินใช้ประจำ

**สำคัญมาก (ต้องเน้นในสไลด์):**
- AI **“แค่แนะนำ” ไม่ส่งเอง** — แอดมินเป็นคนตรวจและกดส่งเสมอ (ปลอดภัย ไม่หลุดตอบผิด)
- ถ้าความรู้ในคลัง **ไม่พอตอบ** → AI จะบอกแอดมินว่า “ขอข้อมูลเพิ่มเข้าสมอง” เพื่อเอาไปเติมคลังความรู้ให้ฉลาดขึ้นเรื่อยๆ

### บล็อก D — ควบคุมด้วยการพิมพ์สั่ง (ภาษาคน)

สั่งน้องวานผ่านแชทได้เลย ไม่ต้องเข้าระบบ:
- **“ปิดแจ้งเตือน Thunder 2 ชั่วโมง”** → ปิดเฉพาะแบรนด์นั้น แล้ว **เปิดเองอัตโนมัติ** เมื่อครบเวลา
- **“สถานะทุกกลุ่ม”** → รายงานว่ากลุ่ม/แบรนด์ไหนเปิด–ปิดแจ้งเตือนอยู่
- **เตือนเช้า–เย็น** กระซิบกันลืมว่ายังปิดแจ้งเตือนอันไหนค้างอยู่

### แถบผลลัพธ์ (ปิดท้ายหน้า 1 — 4 การ์ดสั้น)

- **ไม่พลาดลูกค้า** — ทุกแชทค้างมีคนเห็น
- **ตอบไวขึ้น** — มี AI ร่างให้พร้อมส่ง
- **ไม่ลืมปิดแชท** — เตือนเมื่อค้างเกินเวลา
- **เห็นที่เดียว** — ทุกแบรนด์รวมในกลุ่มเดียว

# หน้า 2 — “Flow การทำงาน” (ไดอะแกรมเส้น)

> ทำเป็น **flow ซ้าย → ขวา** มีเส้น/ลูกศรโยงระหว่างกล่อง + **ลูปป้อนกลับ 1 เส้น** (จากกล่องคำสั่ง กลับไปที่ขั้น “แจ้งเตือน”) สไตล์ไดอะแกรม ไม่ใช่ตาราง

### โหนด (กล่อง) เรียงตามลำดับ

**[ กล่อง 0 · แหล่งข้อมูล ]** (ซ้ายสุด — การ์ดใหญ่)
- 3 แพลตฟอร์ม: **OHO** (รวมแชททุกเพจ/ไลน์) · **Facebook Inbox** · **LINE OA**
- บริการที่เฝ้า: Thunder Bot · Thunder API · EasySlip · EasyCRM · BoostSMS
- *แผนที่จริง: OHO→(Thunder Bot, Thunder API, EasySlip) · Facebook→(Thunder API, BoostSMS) · LINE→(EasyCRM, BoostSMS)*

⬇ **ลูกศร**

**[ กล่อง 1 · เฝ้าดู ]** — ไอคอน “ตา/รีเฟรช”
- บอทล็อกอินเข้าทุกช่องทาง สแกน **ทุก 60 วินาที** ตลอด 24 ชม.

⬇ **ลูกศร**

**[ กล่อง 2 · ตรวจจับ ]** — ไอคอน “เตือน”
- ⏱ ลูกค้ารอเกิน **3 นาที** ยังไม่มีคนรับ
- ⚠ แอดมินรับแล้ว **ไม่ปิดเกิน 15 นาที**

⬇ **ลูกศร**

**[ กล่อง 3 · อ่าน + AI ร่างคำตอบ ]** — ไอคอน “หุ่นยนต์/AI” (กล่องเด่นสีเข้ม)
- อ่านบทสนทนาจริง + แคปหน้าจอ + จำแนกแบรนด์
- AI ร่างข้อความตอบ **โดยดึงจากคลังความรู้บริษัท (KB)** ← *ให้มีเส้นโยงเล็กๆ จากไอคอน “คลังความรู้/ฐานข้อมูล” เข้ามาที่กล่องนี้ เพื่อสื่อว่า “ข้อมูลตอบลูกค้ามาจากที่นี่”*
- ผลลัพธ์: ข้อความพร้อมก๊อป (แอดมินตรวจแล้วส่งเอง)

⬇ **ลูกศร**

**[ กล่อง 4 · แจ้งเตือน ]** — ไอคอน “กระดิ่ง”
- แท็กแอดมิน “เวร” ที่ต้องรับ + แนบรูป + ปุ่มเปิดแชท

⬇ **ลูกศร**

**[ กล่อง 5 · ปลายทาง: กลุ่ม Telegram “monitor” ]** (ขวาสุด — การ์ดใหญ่สีเข้ม)
- ทีมเห็นที่เดียว แยกเป็นห้อง (Topic) ตามแบรนด์:
  - Thunder Bot / Thunder API (2 ห้อง)
  - EasySlip (1 ห้อง)
  - EasyCRM · BoostSMS (2 ห้อง)
  - ห้อง “อย่าลืมปิดแชท” (เตือนปิด)

### เส้นลูปป้อนกลับ (Feedback loop)

**[ กล่องล่าง · สั่งงานด้วยภาษาคน (น้องวาน) ]** → ลากเส้น (เส้นประ) กลับขึ้นไปที่ **กล่อง 4 (แจ้งเตือน)**
- ป้ายกำกับเส้น: **“สั่ง = ปรับการแจ้งเตือนได้ทันที”**
- ตัวอย่างคำสั่ง: “ปิดแจ้งเตือน Thunder 2 ชม.” · “สถานะทุกกลุ่ม”

### (ปลายทางฝั่งคน) หลังได้รับแจ้งเตือน แอดมินทำ 3 อย่าง
กดรับแชท · ก๊อปคำตอบที่ AI ร่างไปส่ง · กดปิดแชทเมื่อจบ
